# System RPG dla RPGon

Ten dokument opisuje autorski system RPG rozwijany dla gry `RPGon`.
Jest to dokument roboczy i będzie rozszerzany o kolejne ustalenia.

## 1. Założenie ogólne

| Element | Opis |
| --- | --- |
| Cel systemu | Zastąpienie mechanik Warhammera nowym, autorskim systemem RPG lepiej dopasowanym do gry |
| Status dokumentu | Dokument roboczy rozwijany wraz z kolejnymi ustaleniami |

## 2. Cechy bazowe postaci

| Cecha | Opis |
| --- | --- |
| Siła | Fizyczna moc postaci, skuteczność w walce wręcz, dźwiganie oraz działania wymagające brutalnej siły |
| Inteligencja | Wiedza, analiza, logiczne myślenie, pamięć oraz zdolność rozumienia złożonych zjawisk |
| Charyzma | Wpływ społeczny, perswazja, blef, przywództwo i ogólne wrażenie wywierane na innych |
| Zręczność | Refleks, precyzja ruchów, skradanie, uniki oraz obsługa narzędzi i broni wymagających dokładności |
| Wytrzymałość | Odporność organizmu, kondycja, zdrowie, wytrwałość i zdolność przetrwania trudnych warunków |
| Szczęście | Cecha specjalna dająca każdemu rzutowi `X%` szans na gwarantowany sukces, gdzie `X` jest równy aktualnemu poziomowi `Szczęścia` |
| Mana | Zasób magiczny wykorzystywany do rzucania zaklęć; można ją zwiększać wyłącznie przy pomocy magicznych kamieni |

## 5. Uwagi projektowe

| Temat | Ustalenie |
| --- | --- |
| Główna cecha umiejętności | Każda umiejętność ma jedną główną cechę bazową |

| Rola `Many` | `Mana` nie jest zwykłą cechą rozwojową; to osobny zasób magiczny związany z zaklęciami |

## 7. Magia i mana

| Element | Ustalenie |
| --- | --- |
| Cecha magiczna | `Mana` |
| Funkcja many | Służy do rzucania zaklęć |
| Rozwój many | `Mana` może być zwiększana wyłącznie przy pomocy magicznych kamieni |
| Rzadkość zasobu | Magiczne kamienie są rzadkie, więc nawet koszt `5` many jest bardzo wysoki |
| Zakres kosztów zaklęć | Większość zaklęć powinna kosztować `1-3` many, a `4-5` many powinno oznaczać zaklęcia bardzo silne albo wyjątkowe |

### 7.1 Zwoje i nauka zaklęć

| Mechanika | Ustalenie |
| --- | --- |
| Nauka zaklęcia | Aby nauczyć się zaklęcia, gracz musi znaleźć odpowiedni scroll |
| Podstawowa szansa nauki | `25%` |
| Modyfikator | Szansa nauki jest zwiększana przez `Inteligencję` |
| Jednorazowe użycie scrolla | Scroll może zostać zużyty jednorazowo do rzucenia zaklęcia, nawet jeśli gracz jeszcze się go nie nauczył |
| Wybór gracza | Po znalezieniu scrolla gracz decyduje, czy próbuje nauczyć się zaklęcia, czy zużywa scroll do natychmiastowego rzutu |
| Rozwój w drzewku zaklęć | Awans do kolejnych zaklęć w ścieżce nie dostaje dodatkowych ograniczeń związanych ze scrollami ani `Inteligencją` |
| Status szczegółowej formuły | Dokładny wzór przeliczania `Inteligencji` na bonus do nauki pozostaje do ustalenia |

### 7.2 Drzewka zaklęć

| Zasada progresji | Ustalenie |
| --- | --- |
| Rozwój czaru | Używanie danego czaru zwiększa biegłość w tym konkretnym zaklęciu |
| Odblokowanie kolejnego poziomu | Kolejny czar w ścieżce odblokowuje się po odpowiedniej liczbie użyć poprzedniego |
| Standardowy próg użyć | `5 x koszt many` poprzedniego czaru |
| Próg dla przejścia z poziomu 1 na poziom 2 | `2 x (5 x koszt many czaru poziomu 1)` |
| Dodatkowe ograniczenia | Na tym etapie nie dodaje się dodatkowych ograniczeń związanych ze scrollami ani `Inteligencją` przy awansie w drzewku |

## 9. Sposób testowania (mechanika rzutów)

### 9.1 Kość i podstawowa formuła

| Element | Ustalenie |
| --- | --- |
| Kość bazowa | `d50` (zakres `1-50`) |
| Formuła testu | `Wynik = d50 + Cecha + Poziom Umiejętności + Momentum + Kreatywność + Przychylność` |
| Próg trudności | Stała wartość zależna od trudności testu |
| Sukces | `Wynik ≥ Próg Trudności` |
| Margines | `Wynik − Próg Trudności` (dodatni = sukces, ujemny = porażka) |

### 9.2 Progi trudności

| Trudność | Próg |
| --- | --- |
| Bardzo łatwa | `15` |
| Łatwa | `25` |
| Przeciętna | `35` |
| Trudna | `45` |
| Bardzo trudna | `55` |
| Niezwykle trudna | `65` |

### 9.3 Modyfikatory

| Modyfikator | Zakres | Opis |
| --- | --- | --- |
| Momentum | `−10` do `+10` | Narastająca seria sukcesów lub porażek. Rośnie o `+1` po sukcesie, maleje o `−1` po porażce |
| Kreatywność | `0` do `+10` | Bonus za kreatywne, taktyczne lub fabularne opisanie akcji |
| Przychylność BN | zmienny | Bonus za pozytywne nastawienie BN-a (dotyczy testów `Charyzmy`) |

### 9.4 Mechanika Szczęścia

| Element | Ustalenie |
| --- | --- |
| Sprawdzenie | Przed rzutem `d50` losowana jest wartość `1-100` |
| Gwarantowany sukces | Jeśli wylosowana wartość ≤ `Szczęście` postaci, test jest automatycznym sukcesem |
| Zastosowanie | Działa na każdy test, niezależnie od trudności |

### 9.5 Walka

| Element | Ustalenie |
| --- | --- |
| Atak wręcz | Test `d50 + Siła + Walka bronią` vs próg trudności |
| Atak dystansowy | Test `d50 + Zręczność + Celność` vs próg trudności |
| Obrażenia | `Siła + Bonus broni − Wytrzymałość celu − Pancerz celu` (minimum `1`) |
| Margines | Dodatni margines udanego ataku dodaje się do obrażeń |
| Unik | Test `d50 + Zręczność + Uniki` vs próg trudności |

### 9.6 Rzucanie zaklęć

| Element | Ustalenie |
| --- | --- |
| Koszt | Zaklęcie kosztuje `Mana` z drzewka zaklęć |
| Test | Nie wymaga osobnego testu — jeśli postać ma wystarczającą manę, zaklęcie jest rzucone |
| Efekt | Zdefiniowany przez konkretne zaklęcie w drzewku |
| Progresja | Użycie zaklęcia daje biegłość; po `5 × koszt many` użyć odblokowuje się kolejny poziom |
